Abstract V2 - Jonathan Gigi, Alfred Antony, Aidan George — Cypress Woods High School

Machine-learning solar-flare forecasts are evaluated on curated benchmarks and routinely report 90–99% accuracy. Whether those scores reflect real-time operational performance on the raw satellite data a deployed system actually consumes has not been tested. We ask whether benchmark scores overstate operational skill, and whether training on live data closes the gap.
Using a RandomForest trained on the SWAN-SF benchmark (True Skill Statistic, TSS, a flare-forecast accuracy measure that isn't skewed by rare events: 0.77), we built an out-of-sample evaluation using live SDO satellite magnetic-field data and NOAA flare records across three solar-cycle phases. At its default settings, the model's TSS fell to 0.35 (2014), 0.64 (2015), and 0.53 (2017) — below the benchmark everywhere, and below chance-adjusted expectations in 2014 and 2017.
Retraining on live data did not close this gap: the overstatement is inherent to the benchmark's evaluation set, not a data-mismatch problem. The main cause was a threshold artifact — the model's default decision cutoff was tuned to optimize F1 score, which collapses as real flares become rarer. Re-tuning that cutoff for TSS instead, using only benchmark data, recovered performance to 0.82–0.84 on unseen years; adding live data on top gained at most +0.02 more.
Auditing our own evaluation also revealed catalog decay: correct flare attribution fell from 94–98% (2013–2017) to just 15% by 2023. We built a fail-closed label-quality gate to guard against this going forward. Plain accuracy is meaningless in this setting — a zero-skill model still scores "99% accuracy."
This work offers a reproducible method for honestly evaluating operational flare forecasts, and a caution against reading benchmark scores as real-world performance.
